Orchestration Layer
🎯 Orchestrator Agent (Chat Agent / Parent LangGraph Graph) Haiku — intent (<500ms) S13 ▶
  • Intent Detection — Haiku classifies user message into agent domain (<500ms)
  • Agent Selection — Routes to specialist via conditional edges
  • Context Routing — Passes only relevant context to sub-agent (smart section extraction)
  • Result Synthesis — Combines specialist outputs for user-facing response
  • Halt Gates — interrupt() at critical decision points
  • Multi-Model Routing — Haiku for Q&A, Sonnet for generation, Opus for complex reasoning
Scope Selection ICP Direction Draft Review Cost Confirmation

LangGraph StateGraph with typed state schema. interrupt() for halt gates. Conditional edges for routing. Replaces monolithic agent_executor.py.

🔍 Research Agent (Coordination Layer) Sonnet S15 ▶

Orchestrates sub-agents for company profiling, contact enrichment, market analysis, and document processing. Parallel fan-out via LangGraph Send().

Company Profiler

Contact Enricher

Market Analyst

Hierarchical delegation: Research Agent itself orchestrates sub-agents. Company Profiler + Contact Enricher can run in parallel. Key insight: enrichment tools are dumb (API calls), enrichment agents are smart (decide what, when, and what to do with results).

📨 Outreach Agent Sonnet S19 ▶

Message generation, personalization, and campaign planning. Uses strategy context + enrichment data for grounded, persona-aware messaging.

generate_message personalize_message create_campaign assign_to_campaign

Consumes strategy document sections and enrichment data as context. Supports per-message regeneration overrides: language, formality (Ty/Vy), tone, custom instruction. Cost estimation before generation.

Cross-Cutting Services
🧠 Memory Layer S15 ▶

Cross-session (RAG): pgvector embeddings of key decisions, preferences, outcomes. Retrieved via semantic search at session start.

Within-session: Floating context window. When history exceeds 15 messages, summarize oldest 10 into ~200 tokens preserving decisions + tool outcomes. Keep last 8 verbatim.

Auto-save: Decisions at halt gates automatically persisted. Agent remembers ICP selections, messaging angles, strategy preferences across sessions.

🔧 Tool Router S13 ▶

Phase-filtered tools: Only register tools relevant to current phase. Strategy phase gets strategy tools, not campaign tools.

Reduction: ~2.5K tokens (all 24 tools) → ~600-1K tokens (6-10 phase-relevant tools) per call.

Pattern: Orchestrator determines intent → selects specialist → specialist has only its 5-8 tools registered.

📜 Prompt Layers S12 ▶
L0 Identity Role, rules, style, language (~800 tok) CACHED
L1 Capabilities Phase-filtered tool descriptions (~1-2K tok) CACHED
L2 Context Phase instructions, section completeness, enrichment (~1-5K tok) DYNAMIC
L3 Conversation Summarized history + recent messages + tool results (~1-4K tok) DYNAMIC

Expected savings: 50-70% input token reduction. From 8-20K per call down to 3-7K with cached static layers. Uses Anthropic cache_control: ephemeral.

📎 Multimodal Pipeline S17 ▶

File ingestion pipeline: Upload → S3 → Format-specific extractor → Summarize → Cache in PG → Inject into agent context on demand.

PDF (Phase 1) Images (Phase 1) HTML (Phase 2) Word (Phase 2) Excel (Phase 3) Video (Phase 4)

Progressive detail: L0 mention (~20 tok) → L1 summary (~300-700 tok) → L2 deep dive (up to 4K tok, on demand via tool).

🔄 Shared State (AG-UI) S13 ▶

STATE_DELTA: Incremental state patches from agent to frontend. Agent updates company data → frontend table updates in real-time without refetch.

STATE_SNAPSHOT: Full state sync on reconnection or page refresh.

Generative UI: Agent sends state patches, frontend renders rich components (tables, charts, approval forms) inline in chat.

🔎 Observability (LangSmith) S13 ▶

Per-node execution traces: duration, tokens, cost. Prompt playground for A/B testing. Evaluation datasets for regression testing. Replay failed conversations.

No paid observability until revenue. Self-hosted LangSmith if feasible, otherwise lightweight custom tracing.

Infrastructure & Data Layer

pgvector extension planned for RAG memory embeddings. 42+ migrations applied.

💰 Token / Credit System S8 ▶

Per-operation cost tracking. Every Claude API call logged: input/output tokens, model, cost, operation type. Per-tenant credit balance. Budget controls with thresholds.

1 credit = $0.001 USD. Users see tokens, admins see raw USD. Cost estimation shown before expensive operations.
